# Памятка по проекту `Первое-Второе`

Этот текст можно передать другому программисту или другому ИИ без дополнительных пояснений.

## Что это за проект

Это статический сайт меню и заказа еды для столовой `Первое-Второе`.

Сайт:
- работает на GitHub Pages;
- хранится в репозитории `pervto/Pervoe-Vtoroe`;
- не имеет своего сервера;
- все действия делает прямо в браузере посетителя;
- открывает заказ через WhatsApp;
- берёт меню из Google Sheets.

Важно: так как это GitHub Pages, здесь нельзя рассчитывать на свой серверный код. Если нужна логика на стороне сервера, её надо делать отдельно через внешний сервис.

## Главный принцип работы

Сайт работает так:
- открывается страница;
- браузер загружает меню из Google Sheets;
- из таблицы собираются блюда, категории и баннеры;
- человек листает меню, открывает блюдо, собирает корзину;
- при нажатии на кнопку заказа сайт собирает текст сообщения;
- дальше сайт открывает WhatsApp по ссылке `wa.me/...` с уже готовым текстом заказа;
- после возврата из WhatsApp сайт спрашивает, удалось ли реально отправить заказ;
- если да, корзина очищается и показывается финальная плашка с благодарностью.

## Где размещён сайт

Сайт рассчитан на GitHub Pages. Это значит:
- сайт состоит из обычных файлов `HTML`, `CSS`, `JS`, картинок и манифеста;
- своего сервера у сайта нет;
- любые проверки времени, загрузка таблицы и другая логика идут из браузера;
- после изменений важно обновлять версии файлов, иначе браузер и service worker могут держать старую версию.

## Откуда сайт берёт меню

Источник меню — Google Sheets.

Сейчас в [config.js](/C:/Users/parsi/OneDrive/Рабочий стол/PV/config.js) есть 3 варианта настройки:
- `csvUrl` — текущая публичная CSV-ссылка;
- `sheetUrl` — обычная ссылка на таблицу;
- `sheetId` и `sheetGid` — прямой способ для более быстрой синхронизации.

Практически:
- если заполнен только `csvUrl`, сайт работает через опубликованную CSV-версию таблицы;
- если заполнить `sheetUrl` или `sheetId`, сайт сможет ходить к таблице более прямым способом и быстрее замечать изменения;
- для этого таблица должна быть открыта на просмотр по ссылке.

## Какие столбцы ждёт сайт в таблице

Обязательные столбцы:
- `Наименование`
- `Цена`
- `Категория`
- `Наличие`

Полезные дополнительные столбцы:
- `Фото 1`
- `Фото 2`
- `Фото 3` и дальше тоже поддерживаются, если назвать как фото;
- `Описание`
- `Вес`
- `Калории`
- `Порядок категорий`
- `Баннеры`

Как это работает:
- `Наличие` управляет тем, показывать блюдо или нет;
- `Порядок категорий` задаёт порядок кнопок категорий;
- `Баннеры` создают верхнюю ленту баннеров;
- фото собираются в галерею блюда.

Что считается наличием:
- значения вроде `да`, `yes`, `true`, `1` считаются наличием;
- всё остальное считается отсутствием.

## Как устроено обновление меню

Сайт не только грузит меню при первом открытии, но и дальше фоном его перепроверяет.

Сейчас логика такая:
- полная загрузка меню идёт при старте;
- дальше сайт регулярно сам перепроверяет изменения;
- более лёгкая проверка отдельно следит за наличием;
- если блюдо пропало из наличия, сайт может убрать его из карточек и из корзины;
- если изменения крупнее, сайт делает полную тихую перезагрузку меню.

Важно понимать ограничение:
- если меню читается через опубликованную CSV-ссылку Google, сама Google-таблица иногда отдаёт изменения с задержкой;
- это ограничение не GitHub Pages, а именно способа публикации таблицы;
- более быстрый режим лучше строить через `sheetUrl` или `sheetId`.

## Как работает заказ

Сценарий заказа:
- человек добавляет блюда в корзину;
- открывает корзину;
- вводит имя, телефон, адрес и при желании комментарий;
- нажимает кнопку заказа;
- сайт собирает текст сообщения;
- открывает WhatsApp;
- сохраняет временную копию заказа в памяти браузера;
- после возврата показывает окно подтверждения;
- если человек подтвердил, что заказ реально отправлен, корзина очищается;
- затем показывается финальное окно благодарности.

## Как работает рабочее время

Сейчас сайт считает рабочим временем:
- с `9:00` до `18:00`
- по часовому поясу `UTC+5`

Проверка идёт не по часам телефона пользователя, а через внешние источники времени.

Сейчас используются:
- `timeapi.io`
- запасной вариант `worldtimeapi.org`

Поведение:
- в рабочее время заказ доступен;
- вне рабочего времени над логотипом появляется закреплённая плашка;
- меню и корзина остаются доступными;
- кнопка отправки заказа отключается;
- если сайт временно не смог проверить время, заказ тоже блокируется, чтобы не было ошибочных заказов.

## Известные технические нюансы

- GitHub Pages не даёт своего серверного времени, поэтому проверка часов работы идёт через внешние сайты времени.
- Google Sheets в режиме опубликованной CSV может обновляться с задержкой.
- В терминале часть русских строк может выглядеть криво из-за кодировки, но в браузере сайт при этом может работать нормально.
- Локальная папка проекта у владельца может быть просто копией без папки `.git`, даже если сам сайт реально хранится на GitHub.

## Изменино:

- Номер версии сайта теперь берётся автоматически из `SITE_VERSION` в `service-worker.js`. Он показывается и внизу сайта, и мелко внизу окна настроек. Текущее значение — `1.23`.
- Помидор у поиска не удалён из проекта, но сейчас выключен через флаг `SEARCH_TOMATO_ENABLED = false` в `index.html`. Чтобы вернуть его, достаточно поменять `false` на `true`.
- Для помидора в проекте уже есть готовая логика: отдельная кнопка, анимация, пульс при нажатии, скрытие при фокусе на поиске и логика слоёв поверх баннеров и шапки.
- Логотип в шапке при нажатии пульсирует.
- На телефонах поиск дополнительно фиксируется при открытии клавиатуры, чтобы поле не уезжало из видимой части экрана.
- Верхние баннеры мягко скрываются при фокусе или вводе в поиск и возвращаются обратно после выхода из поиска.
- Плашка ошибки загрузки меню переделана: теперь это отдельная широкая карточка без сырых технических текстов вроде `fetch is aborted`.
- У фотографий блюд есть двухэтапная загрузка: сначала быстрый маленький preview `sz=w80`, потом основное фото плавно подменяет его через прозрачность.
- Основной размер фото в плитках сейчас `sz=w510`.
- Основной размер фото в открытой карточке блюда сейчас `sz=w800`.
- Для фото блюд включена ранняя предзагрузка: большая версия начинает грузиться уже в момент касания карточки, ещё до открытия окна блюда.
- Для фото блюд добавлен отдельный кеш `PHOTO_CACHE` в `service-worker.js`. При повторном открытии те же фото могут появляться без повторного запроса к Google Drive.
- Если Google Drive не отдаёт картинку, сайт теперь показывает штатную плашку `Фотография блюда скоро появится`, а не синий значок битого изображения.
- При сборке ссылок на фото сайт теперь сохраняет `resourcekey` Google Drive, если он есть в исходной ссылке.
- В `localStorage` дополнительно хранится кэш переводов `pervoe-vtoroe-translation-cache-v2`.
- Логотип автоматически переключается между светлой и тёмной версией в зависимости от выбранной темы.
- Поиск по блюдам теперь всегда идёт по всему меню, даже если у пользователя открыта конкретная категория. Когда поиск очищают, снова начинает работать обычная фильтрация по выбранной категории.
